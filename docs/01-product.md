# 01. Product — 무엇을, 누구를 위해, 어디까지

## 한 줄 목표

> **"지금 누가 뭐 해?"라는 질문이 사라지게 한다.**

팀 업무를 한 화면에서 시각화해 불필요한 확인 커뮤니케이션을 없앤다.

---

## 페르소나

| 항목 | 내용 |
|------|------|
| 대상 | 10명 규모 스타트업 팀리더 |
| 연령대 | 30~40대 |
| 상황 | 슬랙·노션·메일을 오가며 팀원 진행 상황을 수동으로 파악 중 |
| 핵심 불편 | "각자 뭐 하는지 물어봐야만 알 수 있다" |
| 원하는 것 | 별도 커뮤니케이션 없이 팀 전체 업무 현황을 한눈에 파악 |

---

## MVP 범위

### 포함

| 기능 | 설명 |
|------|------|
| 태스크 생성 | 제목·설명·상태·마감 시각 입력 후 저장 |
| 태스크 목록 | 전체 태스크를 상태별로 분류해 표시 |
| 태스크 수정 | 모든 필드 인라인 수정 |
| 태스크 삭제 | 확인 후 영구 삭제 |
| 상태 분류 | `todo` / `in_progress` / `done` 3단계 |
| 마감 시각 | 날짜 + 시간까지 지정 (예: `2026-05-12 18:00`) |
| 라이트/다크 테마 | 헤더 토글 버튼 · `localStorage` 유지 |
| 모바일 반응형 | 360px 기준 최소 너비 보장 |

### 확장 (MVP 이후 — 별도 문서에서 관리)

| 기능 | 비고 |
|------|------|
| JWT 로그인 / 회원가입 | 인증 레이어 추가 |
| 팀 / 멤버 관리 | 다중 사용자 |
| Kanban 보드 | 드래그 앤 드롭 |
| 채팅 / 댓글 | WebSocket (MVP는 폴링) |
| CI/CD 파이프라인 | 자동 배포 |

### 범위 외 (영구 제외)

| 항목 | 이유 |
|------|------|
| 외부 결제 | 서비스 범위 아님 |
| 네이티브 앱 | 웹 MVP 우선 |
| WebRTC | 복잡도 대비 가치 낮음 |
| 외부 캘린더 연동 | 외부 의존성 추가 |
| 파일 업로드 | 스토리지 인프라 필요 |

## UI 톤 — macOS 스타일

| 요소 | 적용 방식 |
|------|-----------|
| 모서리 | 둥근 모서리 (`border-radius: 12px` 내외) |
| 그림자 | 부드러운 레이어 그림자 (`box-shadow` 다단계) |
| 카드 | 반투명 배경 (`backdrop-filter: blur`) |
| 폰트 | 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) |
| 색상 | 라이트: 흰색 계열 배경 + 회색 텍스트 / 다크: 짙은 회색 계열 |
| 인터랙션 | 호버 시 부드러운 전환 (`transition: all 0.2s ease`) |

## 성공 기준

모든 항목을 **직접 눈으로 확인**해야 MVP 완료로 선언한다.

| # | 기준 | 확인 방법 |
|---|------|-----------|
| 1 | 새로고침 후 데이터 유지 | 태스크 생성 → F5 → 목록에 존재 |
| 2 | 360px에서 레이아웃 안 깨짐 | 브라우저 DevTools → 360px 너비 설정 |
| 3 | API 응답 200ms 이하 | Network 탭 → 각 요청 응답 시간 확인 |
| 4 | CRUD 4종 화면 동작 | 생성 → 조회 → 수정 → 삭제 순서대로 실행 |
| 5 | 테마 토글 작동 | 토글 클릭 → 테마 전환 → 새로고침 후 유지 |
