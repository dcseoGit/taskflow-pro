# 02. Specs — 데이터 모델 · API · 화면 명세

## Task 모델

### 필드 정의

| 필드 | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| `id` | INTEGER | — | 자동 증가 | 고유 식별자 (PK) |
| `title` | VARCHAR(200) | ✅ | — | 태스크 제목 |
| `description` | TEXT | ❌ | `""` | 상세 설명 |
| `status` | VARCHAR(20) | ❌ | `"todo"` | 진행 상태 |
| `due_at` | DATETIME (UTC) | ❌ | `null` | 마감 시각 |
| `created_at` | DATETIME (UTC) | — | 서버 자동 | 생성 시각 |
| `updated_at` | DATETIME (UTC) | — | 서버 자동 | 최종 수정 시각 |

### status 허용값

| 값 | 의미 |
|----|------|
| `todo` | 할 일 (기본값) |
| `in_progress` | 진행 중 |
| `done` | 완료 |

---

## 유효성 검증

| 필드 | 위반 조건 | 응답 |
|------|-----------|------|
| `title` | 누락 또는 빈 문자열, 201자 이상 | `400 Bad Request` |
| `status` | `todo` / `in_progress` / `done` 외 값 | `400 Bad Request` |
| `due_at` | ISO 8601 형식 불일치 (예: `"2026-05-12 18:00:00"` 또는 `"2026-05-12T18:00:00Z"`) | `400 Bad Request` |
| `id` (경로) | DB에 존재하지 않는 id | `404 Not Found` |

**`due_at` 허용 형식 예시**

```
2026-05-12T18:00:00Z          ← UTC 명시
2026-05-12T18:00:00+09:00     ← 타임존 오프셋
2026-05-12T18:00:00           ← UTC로 간주
```

## REST API 명세

### 공통

| 항목 | 값 |
|------|-----|
| Base URL | `http://localhost:8000` |
| Content-Type | `application/json` |
| 인증 | 없음 (MVP) |
| Swagger UI | `http://localhost:8000/docs` |

---

### 1. `POST /tasks` — 태스크 생성

**요청 바디**

```json
{
  "title": "디자인 시안 검토",
  "description": "피그마 링크 확인 후 코멘트",
  "status": "todo",
  "due_at": "2026-05-12T18:00:00Z"
}
```

**응답 `201 Created`** — 생성된 태스크 전체 필드 반환

```json
{
  "id": 1,
  "title": "디자인 시안 검토",
  "description": "피그마 링크 확인 후 코멘트",
  "status": "todo",
  "due_at": "2026-05-12T18:00:00Z",
  "created_at": "2026-05-14T10:00:00Z",
  "updated_at": "2026-05-14T10:00:00Z"
}
```

---

### 2. `GET /tasks` — 태스크 목록 조회

**요청**: 바디 없음

**응답 `200 OK`** — `description` **제외**, `created_at` 기준 내림차순

```json
[
  {
    "id": 1,
    "title": "디자인 시안 검토",
    "status": "todo",
    "due_at": "2026-05-12T18:00:00Z",
    "created_at": "2026-05-14T10:00:00Z",
    "updated_at": "2026-05-14T10:00:00Z"
  }
]
```

> 태스크 없을 때: `[]` 반환

---

### 3. `GET /tasks/{id}` — 태스크 단건 조회

**응답 `200 OK`** — `description` **포함**, 전체 필드 반환

```json
{
  "id": 1,
  "title": "디자인 시안 검토",
  "description": "피그마 링크 확인 후 코멘트",
  "status": "todo",
  "due_at": "2026-05-12T18:00:00Z",
  "created_at": "2026-05-14T10:00:00Z",
  "updated_at": "2026-05-14T10:00:00Z"
}
```

**응답 `404 Not Found`**

```json
{ "detail": "Task not found" }
```

---

### 4. `PUT /tasks/{id}` — 태스크 수정 (부분 수정)

변경할 필드만 포함. 누락된 필드는 기존 값 유지.

**요청 바디**

```json
{
  "status": "in_progress",
  "due_at": "2026-05-13T09:00:00Z"
}
```

**응답 `200 OK`** — 수정된 태스크 전체 필드 반환 (`updated_at` 갱신)

**응답 `404 Not Found`**: `{ "detail": "Task not found" }`

---

### 5. `DELETE /tasks/{id}` — 태스크 삭제

**응답 `204 No Content`** — 바디 없음

**응답 `404 Not Found`**: `{ "detail": "Task not found" }`

---

### 공통 에러 응답

| 상태 코드 | 발생 조건 |
|-----------|-----------|
| `400` | 필드 유효성 검사 실패 |
| `404` | 존재하지 않는 id |
| `422` | Pydantic 파싱 실패 (타입 불일치 등) |
| `500` | 서버 내부 오류 |

## 화면 명세 (CRUD 4종 UI)

### 1. 태스크 추가 — 폼

| 요소 | 상세 |
|------|------|
| 입력 필드 | `title` (필수) · `due_at` datetime-local 입력 · `status` 드롭다운 |
| 제출 버튼 | "추가" 클릭 → `POST /tasks` → 성공 시 목록 갱신 |
| 에러 표시 | title 비어있으면 제출 막고 인라인 오류 메시지 |
| 위치 | 페이지 상단 고정 폼 또는 플로팅 버튼 → 폼 펼침 |

---

### 2. 태스크 목록 — 카드

| 요소 | 상세 |
|------|------|
| 카드 단위 | 태스크 1개 = 카드 1장 |
| status 배지 | `todo` 회색 · `in_progress` 파란색 · `done` 녹색 |
| 마감 표시 | `D-N HH:MM` 형식 (예: `D-3 18:00`) |
| 마감 경고 | D-1 이하: 주황색 · 마감 지남: 빨간색 |
| 정렬 | `created_at` 내림차순 (최신이 위) |
| `description` | 목록 카드에서 미표시 (단건 조회 시 표시) |

**D-N 계산 기준**

```
D-N = ceil((due_at - 현재시각) / 1일)
D-0: 오늘 마감
D+N: 마감 지남 → 빨간색 표시
```

---

### 3. 태스크 수정 — 모달

| 요소 | 상세 |
|------|------|
| 트리거 | 카드 클릭 → 모달 오픈 + `GET /tasks/{id}` 호출 |
| 표시 필드 | 전체 필드 표시 (`description` 포함) |
| 수정 방식 | 모든 필드 편집 가능 · 변경된 필드만 `PUT /tasks/{id}` 전송 |
| 저장 버튼 | "저장" 클릭 → 응답 성공 시 모달 닫기 + 목록 갱신 |
| 닫기 | 모달 외부 클릭 또는 ESC 키 |

---

### 4. 태스크 삭제 — 확인 후 삭제

| 요소 | 상세 |
|------|------|
| 트리거 | 카드 내 휴지통 아이콘 클릭 |
| 확인 단계 | 인라인 확인 메시지 + "삭제" / "취소" 버튼 |
| 실행 | "삭제" 클릭 → `DELETE /tasks/{id}` → `204` 응답 시 카드 제거 |
| 취소 | "취소" 클릭 → 원래 카드 상태로 복귀 |
| 실패 처리 | `404` 응답 시 "이미 삭제된 태스크입니다" 메시지 표시 |
